Boe-Bot King

Autonomous line-following robot using QTI infrared sensors

Project Overview

Boe-Bot King is an autonomous mobile robot designed to follow a predefined track using dual QTI infrared sensors. The project emphasizes reactive sensor-based decision logic, structured motion control, and reliable recovery behaviors during navigation.

Objective

Design and implement a fully autonomous line-following robot capable of navigating a constrained obstacle course while maintaining accurate path tracking and recovering from collisions or loss of line detection.

Hardware Configuration

  • Boe-Bot mobile chassis
  • Dual QTI infrared reflectance sensors
  • Whisker contact sensors
  • Piezoelectric speaker and status LEDs
  • Onboard microcontroller (BASIC Stamp)

Behavior Decision Mapping

Boe-Bot King uses a deterministic sensor-to-action mapping strategy. Each motion command is selected directly based on the real-time state of the QTI and whisker sensors, ensuring fast response and predictable behavior.

Left QTI Right QTI Whiskers Robot Action
No Line No Line Clear Move Forward
Line No Line Clear Turn Left
No Line Line Clear Turn Right
Line Line Clear Reverse
Any Any Triggered 360° Recovery Turn

This rule-based mapping avoids complex state machines and simplifies debugging during iterative hardware testing.

Control Logic & Behaviors

The robot operates using a reactive control strategy in which motor commands are selected based on real-time sensor readings. Each sensor condition maps directly to a motion behavior.

  • Forward: No line detected by either QTI sensor
  • Left Turn: Line detected by left QTI sensor
  • Right Turn: Line detected by right QTI sensor
  • Reverse: Line detected by both QTI sensors
  • 360° Turn: Triggered by whisker collision

Software Architecture

The control program is organized into modular subroutines, each responsible for a specific motion behavior. Sensor states are evaluated continuously, and the appropriate subroutine is invoked without blocking execution.

Pulse counters were used to regulate turning angles and prevent over-rotation, while an onboard LCD provided real-time feedback during debugging and testing. the picture below shows a snippy of the LCD program used.

Testing & Performance

Through iterative testing and calibration, the robot consistently completed the obstacle course. Proper sensor alignment and ground clearance proved critical to maintaining reliable line detection.

Engineering Challenges

  • Inconsistent QTI sensor readings → sensor height adjusted using washers
  • Over-rotation during turns → calibrated pulse counters
  • Software logic errors → extensive on-hardware debugging

Lessons Learned

This project reinforced the importance of precise sensor placement, modular embedded software design, and iterative hardware–software integration in autonomous systems.
